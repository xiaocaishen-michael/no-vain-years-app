### 二次细化(对话过程中追加)

- **顶 nav 左**: ≡ 三横线(**无 99+ badge**),点击左→右滑出浮层 — 浮层后置 spec
- **顶 nav 中**: 空(原图「+ 添加状态」不需要)

- **VIP 标记 + 徽章 + ID 显示**: **删**(M1 没 billing / 反枚举 SC-008)
- **关注 / 粉丝**: **保留**,固定全局假数字(如 N 关注 / N 粉丝),**不可点**;互动模块(真功能)**不在本批次**

- **板块视觉风格**: 上沉浸(photo blur 背景)→ 下干净白底,slide tabs underline 强切换 — PHASE 2 mockup 落地

## 02-left-drawer — 顶 nav 左浮层决策

参考图是 NetEase 浮层全貌(含 12+ 个 list 项)。用户决策:

**保留**(浮层顶部条):

- 头像 + 用户名 + `>` 进入 profile 入口
- 扫码图标(顶部右)

**清除**(全部):

- 麦克风图标
- 「黑胶 VIP 特权」banner
- 「双人空间」banner
- 中间所有 list 项(我的消息 / 云贝 / 装扮 / 创作者 / AI 写歌 / 最近播放 / 定时关闭 / 商城 / 云村有票 / 云推歌 / 我的客服)
- 底部「设置 / 更多」两按钮

**注**:list 项 → **后续单起 spec**,本批次顶 nav `≡` 点击行为按 ADR-0017 占位 UI 边界 → **disabled / toast 占位**。

> 扫码用途备忘(后置 spec 输入): 扫好友 / 扫笔记 / 登录其他设备。

## 03-settings — 设置页决策

参考图是 NetEase 设置页(含播放音质等音乐 app 专属功能)。用户决策:

**保留**(4 个 list 项 + 1 组按钮 + 页脚):

- 第 1 card: 账号与安全 >
- 第 2 card: 通用 > / 通知 > / 隐私与权限 > / 关于 >
- 第 3 card: 切换账号 / 退出登录
- 页脚:《个人信息收集与使用清单》《第三方共享清单》

**删除**(音乐 app 专属):

- 播放与下载 / 音质 / 存储空间 / 未成年人模式

### 子项本批次范围

| 项                                           | 本批次                                                                                  |
| -------------------------------------------- | --------------------------------------------------------------------------------------- |
| 账号与安全 >                                 | ✅ 跳子页(真功能)                                                                       |
| 通用 / 通知 / 隐私与权限 / 关于              | **disabled 占位**,后续单起 spec                                                         |
| 切换账号                                     | **disabled 占位**(M1 单设备,多账号是 M2+)                                               |
| **退出登录**                                 | ✅ **真功能** — 调 `POST /api/v1/auth/logout-all`(server 已有)+ 清本地 token + 跳 login |
| 《个人信息收集与使用清单》《第三方共享清单》 | ✅ **本地静态 placeholder 页**(法务后续补内容)                                          |

> 原计划「退出登录」需新增 server endpoint,摸底后发现 **`logout-all` + `packages/auth#logoutAll()` 都已落地**,UI 直接调用即可,server 端 0 工作量。

## 04-account-security — 账号与安全页决策

参考图是 NetEase 账号与安全页(含安全系数 banner / 多种 list 项)。用户决策:

**保留**(标题 + 5 list 项):

- 标题「账号与安全」(**无 ID 行**)
- 手机号 / 实名认证 / 第三方账号绑定 / 登录设备与授权管理 / 注销账号 / 安全小知识

**删除**:

- **密码**(原始注释 / phone-sms-auth 是无密码登录)
- 顶部客服图标 🎧(M1 没客服系统)
- 「账号安全系数良好」banner + 4 盾牌(评估系统不存在,假装显示是误导)
- `ID: <numeric>` 行(account.id 是 BIGINT,反枚举防御 SC-008 明示禁用,M1 没 short code)
- 「找回账号」(语义与 phone-sms-auth 重合 — 短信验证即登录)

### 子项本批次范围

| 项                                                          | 本批次                                                         |
| ----------------------------------------------------------- | -------------------------------------------------------------- |
| 手机号 mask 显示(`+86 1XX****XXXX`)                         | ✅ 显示(读 `/me`)+ 点击进 **只读 detail 占位页**               |
| 实名认证 / 第三方账号绑定 / 登录设备与授权管理 / 安全小知识 | **disabled 占位**,后续单起 spec                                |
| **注销账号**                                                | ✅ **真功能** — 跳 spec C(`delete-account-cancel-deletion-ui`) |

### 解封 UI 的位置

> 用户初衷:「账户与安全 → 注销 / **解封** / 三方绑定 等等按钮」。
>
> **澄清**:**解封 = cancel-deletion**,语义是 FROZEN 用户**登录时**取消注销 — UX 入口在 phone-sms-auth login flow 拦截弹窗,**不在「账户与安全」list 里**。这是 spec C 的另一半。

## SDD 拆分(供 spec.md 引用)

本批次共 3 个 app 端 spec(线性依赖):

```text
A. my-profile (本 spec)
   ├─ (tabs) group + 4 个 tab 接入
   ├─ profile 页(顶 nav 三 entry / hero / 关注粉丝 / slide tabs 骨架)
   └─ AuthGate 改造(默认进入「我的」 tab)
        ↓
B. account-settings-shell
   ├─ 设置 list / 账号与安全 list
   ├─ 退出登录(调 logout-all)
   ├─ 法规静态 placeholder 页
   └─ 各 disabled 子项
        ↓
C. delete-account-cancel-deletion-ui
   ├─ 注销账号 UI(SMS 触发 + 6 位码 + 双确认)
   └─ login flow 解封拦截弹窗
```

server 端工作量: **0**(`/me` / `logout-all` / `delete-account` / `cancel-deletion` 全部已落地)。
